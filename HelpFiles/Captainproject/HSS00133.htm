Track Master Maintenance Screen

This Screen allows the System Administrator to manage all of the individual tracking tasks/questions that are available in the Client Tracking Screen for Users to work with.  Access to this screen should be limited to very few Users.  If you have all Users adding in new questions or updating existing questions there will be confusion and frustration.  One User could potentially create a question to use and then another User could immediately change it.

Assigning the role of maintaining this area of the system is best if only one or two Users do it. 

Setting up the Master Track and Components is crucial for an agency that is stirving for accurate tracking and reporting.  The Master Track will contain a User-defined series of tasks that will be available to work with.  The main objective is to collect all the necessary information for the PIR.

There is a Master Track and then Component Tracks you will interact with.  Think about it this way - The Master Track is one group of questions that included EVERY task/question in the system.  One master file with everything in it.  Components act as a filter to eliminate looking at tasks that are not relative to what the User is reviewing. 

Additional Component Tracks can be created (so that specific content area specialists don't have to look through the full Master list to find what they want to work with).  For example, a User focused on Health will find their experience less cumbersome if they don't have to look through Family Services tasks or  Education/Disabilities tasks.  Another example Users can benefit from Component Tracks is Health/EPSDT tasks.  That User will be glad to be able to call up the specific tasks to work with for EPSDT without seeing anything else. 

Adding a NEW task/question to the Track Master Maintenance Screen

Locate and click the  button at the top of the screen to access the working screen to add a new question into the system.

This is the screen that opens.  The top of the screen identifies the User is in add mode.

  • Track is a unique code that you assign to each description
  • Description is where you add a brief description of the question any User will see in the Client Tracking Screen.  It should be meaningful to assist Users in task selection when they are updating child applications with information
  • Owner 1, 2 and 3 fields allow you to assign the specific task to one or more component groups.  

The grid box in the middle of the working screen allows the User to assign the task to different FUND supports and set different requirements to each FUND.  Let's look at the columns in the grid box and how they work.

  • Each FUND has it's own separate row.  You do not have to interact with all rows if you have no need.
  • Intake/Entry is the timeframe that you want the system to interact with in relation to when the task should be completed.  This is automatically calculated when selected and all Users can see the benchmark dates in place.
  • Entry Days is the specifc day from point of entry that the system should use.
  • Every Year should be checked if you are managing the task every year.
  • Next Action is checked if there is a 2nd task that comes immediately following the task you're working with.  
  • Next Task - select the next task that should be connected to the task you're working with.
  • Next Days - type in the number of days for when the next task should be completed. 

Editing an existing task/description in the Track Master Maintenance Screen

Locate and click the  button at the top of the screen to access the working screen to add a new question into the system.

This is the screen that opens.  The top of the screen identifies the User is in edit mode.

Use this working screen to make adjustments to existing tasks.

Task Configuration in the Track Master Maintenance Screen

Locate and click   in the bottom left hand corner of the screen to open the working screen to set individualized configurations for the task you are adding.  You can also update these configurations for an existing task any time you would like. 

   

Green checking the enable and require columns will set a control in place for Users to interact with specific fields in the Client Tracking Screen.
