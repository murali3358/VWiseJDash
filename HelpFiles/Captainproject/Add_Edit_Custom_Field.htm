Add a Custom Field to a specific hierarchy in the Fields Control Maintenance Screen

When you access the Custom Fields Description Tab you will find a list box of the custom fields that exist in the system.  When you highlight one the bottom section of the Tab will display how the custom field is designed. 

  • Select the Screen Name wehre you want to design the new custom field.
  • Locate and Click Add in the main Toolbar for the Tab to access the working screen to add a new custom field to the system.

Let's look at an image of the add screen where we can see the bottom area of the screen now has blank screens waiting for you to design the new custom field. 

  • Sequence - the order sequence that the custom fields are displayed in.  A sequence number can only be used once. 
  • Type - The type of field you want designed. 
  • Active Check Box - if this box isn't checked Users cannot see the custom field in order to use it.  This is often used when you are in the middle of designing everything.  You leave the box unchecked until you have everything settled then you check the box so Users can find it and use it.
  • Access - Select the correct option for which household members Users will be able to respond to the question. 
  • Abbreviation -  This is an abbreviation of the Question Description. 
  • Ques Desc - Question Description that will be displayed for the User to see. 
  • Code -  this area on the right-hand side will change depending on the TYPE of custom field you build.  If you're building a drop down field it will ask you to design in the responses you want the Users to select from.

Once you completed everything you need for the new custom field, click SAVE. 

Changed your mind?  Click CANCEL and you can start again.

You can also select any existing custom field and click the EDIT  Button in order to make any changes/updates you need.  Updating a custom field will impact every application that has already used the field. 

You should NEVER delete a custom field.  If you don't want Users to be able to access the custom field, uncheck the Active Check Box.  While you may not want to continue having Users respond to a particular question, they might have lots of times!  You may still want to include that information in a report.  So, uncheck the Active Box - don't delete!
