PIR Logic Association Screen

This screen allows the System Administrator to create counting logic associations that the System will use when counting individual tasks for specific PIR questions.

 

  • Any PIR template question with an H in the PIR column is a "hardcoded" question. No counting logic can be associated to the question. The Hard Coded Options sub panel will allow some modification.
  • Any PIR template question with a T indentifies that logic from a task in client tracking can be associated to this question.
  • Any PIR template question with an I identifies that the counting logic is driven from data already collected in Client Intake. The System Administrator can create a logic association for these questions from a task in client tracking if desired.
  • Questions Description follows the structure of the PIR Report released by the federal government. 

Program Year Field loads to the default program year.  If you want to resource prior program year data you must select the year that you want to work with so the system knows to resource data in a different program year.  This allows you to resource prior year data whenever necessary.  In the example below the system is locked on program year 2012. 

Select the section of the PIR that you want to work with in the Section field example below.

The template PIR Report loads in the GridBox, question by question. You will then be able to assign specific tracking questions in your individualized tracks for the System to monitor and deliver counts on the PIR Counting Tool Screen.

The HS Fund and EHS Fund radio button determine which template report option you will be viewing. 

Add logic to a question in PIR Logic Association Screen

Select the question that you want to add logic assocation to.  (Left Mouse Click to highlight the question BLUE)   Making this selection tells the systems you want to work with that question.

After selecting the question, click the  button at the top of the PIR Logic Association Screen tab in order to access the working screen to add in a new logic assocation to a specific question in the PIR Counting Tool template. 

  • Type - the System will lock in on the area of the system where the tracking data is located.  (When a question is hard coded this option will not be available as the system controls all the logic and the User is not able to change or update that.)
  • Intake Field - select the field that you want to system to resource to find the data records you want to associate.
  • Group - You can have more than one group of logic for the system to work with.  Each group will function separately within its own logic.
  • Conjunction - this determines if the system uses logic to connect more than one question in a string using "and" and "or". 
  • Condition - this field allows you to select the specific question field that the system will use to count with.
  • Year - the field allows you to select one or more years for the system to resource data records.
  • Response - check the box and select the specific response for the system to resource data records.  If more than one response should be looked at, you will want to add a logic line for each response and use the conjuction "and" and "or" as appropriate. 
  • Select Date Key - this allows you to select a specific response field in Client Tracking for the system to resource data records.  It acts as an additional filter in the logic.  The PIR Counting Tool will ignore any data records that do not satisfy the selected field.
  • Valid Date - check this box if you want the system to look at any date that qualifies in the selected date key field
  • Date Range - as opposed to any date, checking this box will allow you to set a date range filter that you want to system to work with when resourcing data records.
  • From Date - the earliest date you want the system to use when resourcing data records.  (1/1/2012 - 3/31/2012)
  • To Date - the oldest date you want the system to use when resourcing data records.  (1/1/2012 - 3/31/2012)
  • click SAVE 

You can update/change the logic assocation for any question any time or as many times as you would like.  Sometimes you add in a new tracking question that needs to be brought into the logic assocation part way through a year.  Once the logic has been edited the system will use that saved logic association when resourcing data records.

Select the question that you want to edit logic assocation to.  (Left Mouse Click to highlight the question BLUE)   Making this selection tells the system you want to work with that question.  If there is more than one logic association make certain that you highlight the specific logic assocation row you want to work with.
