Use the Next Year Preparation Report Screen to roll over Intakes into the next program year.  The system is partitioned by program year.  Once the year is done and ALL attendance has been posted the System Administrator can roll over the system to the next program year. 

That fundamental approach above shared, because it is the BEST approach, the System Administrator can rollover the system without ALL attendance postings completed in the year that is ending.  What that creates is the need for ALL Users to change the year in the system to post attendance for the correct year.  You see, once the system rolls over, the default year changes to the next year.  A User can, however, push the system back to the previous year and post attendance until it is completed. 

Let's understand the template screen you need to use:

 

In the top section of the screen you will find various filters that you can use to tell the system what you want in place when the rollover happens.  Note the Head Start Cutoff Field is the only required field.   Just because the other fields are not required doesn't mean they aren't important! 

  • Report Sequence -  when the system completes the roll over action a report will be generated to tell the User which Intakes rolled over.  This filter establishes the order of the Intakes in that report.
  • Enroll Status -  Both is the deafult selection.  You can change it to enrolled or not enrolled.  Each filter will produce a different list of Intakes.  Both ensures the system picks up ALL qualifying Intakes.
  • Children - Both is the default, children and repeaters are additional filters.  Sometimes programs want to roll over by category.  This filter will allow that process.  If you want to do just one rollover, select both.
  • Roll Over - This filter is awesome, because it allows you to run a list selecting Produce List Only filter and double check what's going to happen.  Until you run this report with the filter Rollover Data to the New Year, nothing happens other than the system genertating a list.  Selecting Rolover Data to the New Year will have the system completed the entire rollover (based on all filters selected in this screen) and the results are permanent.  The next program year will be built and in place for Users to work in.
  • Head Start Cutoff -  the number you place in this box tells the system was age (by year) to stop at when completing the rollover.  If you put 5 in this box, any Intake with HS Funding where the child in the primary client position is 6 or older will not rollover to the next year.  This filter allows the system administrator to limit the Intakes that are in the next year.  It should be shared that you can roll over ALL Intakes into the next without creating any problems.  Some System Administrators just like to eliminate any child that would never be served in the program in the next year. 
  • Headstart/DayCare Cutoff - this box functions the same as the Head Start Cutoff box with the exception of the funding.  Children that are tagged for both HS and DC will be part of this age cut off.
  • DayCare Cutoff -   this box functions the same as the Head Start Cutoff box with the exception of the funding.  Children that are tagged for DC will be part of this age cut off.
  • Rollover Active Enrollment Transactions - IMPORTANT NOTE***  This last filter determines several things that the system will do.  Take your time and think through your options.  When you first check this box the working screen below opens:

This Funding Selection box is where you select which FUNDS you want the system to lock in on for the rollover.  

  • To select the FUND(s) left mouse click the cell directly to the left of the CODE column. 
  • You will see the green check mark appear in the cell. 
  • Green check marked FUND(s) are included in the rollover, unchecked FUND(s) are not.
  • You can also use Select All and Unselect ALL as needed. 
  • Click OK to lock in on your selections and return to the main screen.

Notice now the main screen is completely different now in this section.  You have another decision to make before generating a list of Intakes you can use to complete the rollover. 

  • The Rollover Funding Sources button will allow you to adjust the selections you previously made.
  • Keep Existing Enrollment Dates - ensures that when the system rolls over all active enrollment records, the original enrollment date attached to each child will stay intact. 
  • Enter Enrollment Date - allows the System Administrator to CHANGE the original enrollment date on the Intakes in the rollover.  Be Careful!  Think this through!  The date your select in this field will become the enrollment date for ALL Intakes in the rollover. 

Once you have all the filters set in place as you want them, you can click the GENERATE APPS button.  This system will then present in the grid box ALL qualifying Intakes.  You then select ALL or select the Intakes you want for the rollover.
