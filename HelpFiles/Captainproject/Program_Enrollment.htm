The Program Enrollment Screen allows you to manage a list of clients that are in your program and also potential referrals into your program identified in the SIM (Service Integration Matrix). 

The top section of the screen delivers an accounting of the number of clients you have interacted with.  They could be enrolled, wtihdrawn, be on the wait list, denied or pending.  It gives the User an accurate look at the clients that you are working with, or have worked with, in the program enrollment year.

The program enrollment year is set in program definition by your System Administrator.  Activity for all Intakes must fall within the defined program year.  The program year definition can be changed/adjusted at any time.

The top section of the screen presents current counts in the columns for each program displayed.  The bottom section of the screen, based on the SHOW, AS OF DATE, and STATUS filter selections will display the Intakes that qualify. 

SHOW, AS OF DATE, and STATUS fields allow the User to control the Intakes that load in the grid box in the bottom section.  

  • Show allows the User to load only clients with an exisiting Status Record, or load in all the clients that currently have no status record.
  • As Of Date looks at the date for the status and load the condition of the intake based on the filtered date.
  • Status allows the User to load the grid box with a specific status list.

 Add System Referrals (from the SIM) to the selected program hierarchy

Adding system referrals to the program hierarchy is the way the system goes out and gets the referrals that were sent in the SIM.  This must be completed in order to be able to enroll a client.  Think of it as the way the system goes out and gets the referrals that were sent so that they will show up in this working screen. 

Click the  Button to access the working screen below.
  • Sort By - filters how the names are displayed in the grid box
  • Status - filters the intakes that will appear in the grid box
  • Refer Date and To - set the date range for the records you want the system to work with.
  • First Name and Last Name - these fields allow the User to search for a specific record.
  • Status Date and Status - these fields are used when creating a status record.  The selected date and status will be saved with each selected record. 

Once you have gone out to the SIM and found referrals they will load in the grid box.  See an example image below.

 

The data for the qualifying records loads in the grid box.  We can now see:

  • referral date
  • program that sent the referral (in the SIM)
  • application # (from the referral program)
  • name of the primary client in the qualifying Intake
  • referral status
  • application number in the hierarchy the User is working in.  If this is blank that confirms that the system did NOT find the referred client in the current hierarchy.

Note now there are two new buttons at the bottom of the working screen.

DRAG APPLICANT - allows the User to draw the referral into the hierarchy they are working in.  The system will identify the application in the working hierarchy is a match (by looking at SSN, DOB and NAME).  If the system finds a client record that matches the referred client the system will synchronize the records.  If it doesn't find a match you will be able to drag the applicant into the working hierarchy.  The system will assign an application number.

CREATE STATUS RECORD - by assigning a status date and a status from the drop down field this button will create the new record.  The referral status column will load with the status the User assigned when creating the record.
