Poverty Switches Tab in the Program Definition Screen

Client Intake Switches

Income Verification Message:  No income verification message.

Income Type Only: At the Income Verification Screen, if this box is checked the User will only be able to enter Income Types. No numerical values will be accepted.

Weekly: Controls how weekly intervals in Income Details are annualized. If unchecked the System will get the average of the weekly buckets and multiply by 13. If checked the System will take the average of 4 buckets * 12 *4.33

Example unchecked: 1) 100 2) 200 3) 300 4) 400 =$1,000 * 13 = $13,000.

Example checked: 1) 100 2) 200 3) 300 4) 400 = $1,000 /4 = $250 weekly average * 12 = $3,000 * 4.33 = $12,990.

Include Income Verification:  check box to include income verification.

Prohibit Duplicate Members: Check this box to ensure that at the Client Intake Screen a User cannot add an applicant or a household member that already exists.

Prohibit Duplicate SS#: Check this box to enable to system to monitor duplicate SS# data entry for applicants. 

Federal OMB: If you check this box this will make the Federal OMB % a required field in calculating eligibility in the Income Verification Screen. Remember if you check this box off you will need to define the Federal OMB Guidelines in the Table. 

HUD: If you check this box this will make the HUD % (Housing Urban Development) a required field in calculating a households HUD poverty percentage when verifying income in the Income Verification Screen. Remember if you check this box you will need to define the HUD Guidelines in the Table.

CMI: If you check this box this will make the CMI % (County Median Income) a required field in calculating a households CMI poverty percentage when verifying income in the Income Verification Screen. Remember if you check this box you will need to define the CMI Guidelines in the Table.

SMI: If you check this box this field will make the SMI % (State Median Income) a required field in calculating a households SMI poverty percentage when verifying income in the Income Verification Screen Remember if you check this box you will need to define the SMI Guidelines in the Table.

Edit Intake Date: This is a control for the Client Intake Screen. When start and end dates are locked in Program Definition a User cannot enter an Intake Date outside of that program data range.

Edit Zip: If checked this will check the zip code that the User is entering in Client Intake with Zip Code Maintenance. If the zip code is not in the Zip Code table the User will not be able to save the record until a zip code from the Zip Code table option is selected. If not checked the User would be able to enter any zip code and if the zip code is not in the Zip Code table the User would need to enter the City and State as these fields would be cleared.

Edit Address: This feature is used if your agency loads an external Voter Registration file, which the system uses to validate the address.

Auto Inactive: Functions when you draw an existing application into a new one. If any of the members were inactive creating a new application will activate them.

Edit Case Type: At the Client Intake Screen if this box is checked the User can only change the Case Type in ADD mode. No change to Case Type can be made in CHANGE mode.

Uom: checking this box will automatically calculate the # of units that appear in the Service/Activity posting screen. 

Secret Program: Check this box if access to all data records in this hierarchy should be limited to only specific program Users.  In the MainMenu Search Screen these applications will not be present for review.  Often times there is a need for a highly confidential hierarchy. 

SS # Reason:   If the User uses the PgDn key to have the system auto assign a Pseudo SS#, checking this box will present the SS# Reason field to select the reason why a SS# wasn't entered. 

Juvenile Age From and To: This feature will tell the System for a specific program what age range is considered Juvenile. When a User launches the DASHBOARD there will be a check mark in the Juvenile check box if the applicant is between the age ranges as defined in the Program Definition Screen. The date to use to calculate the age is the intake date from the Client Intake Screen.

Senior Age From and To: This feature will tell the System for a specific program what age range is considered Senior. When a User launches the DASHBOARD there will be a check mark in the senior check box if the applicant is between the age ranges as defined in the Program Definition Screen. The date to use to calculate the age is the intake date from the Client Intake Screen.

SIM Switches

Threshold Value:  The value entered in this field will appear in the SIM (Service Integration Matrix) screen in the Threshold column of the Department/Program List box. The purpose is to assign a threshold point value to maximize referral eligibility for program services/enrollment. Cummulative client responses values that do not meet or exceed the assigned threshold value will not be identified as a potential referral.

System/Manual:   This is used in conjunction with the Threshold Value field. After the User responds to the custom questions used to determine program eligibility presented in the Service Integration Matrix (SIM) window and the total points meet or exceed the threshold, an automatic referral of the client will be made to the program (SYSTEM). Referral management is now in PRESUMPTIVE ELIGIBILITY mode. (or) The User will manually review question responses and has to make a subjective referral to any/all program options.

Points Method: If this box is checked the sum of the points in the Eligibility Criteria Screen must exceed the established threshold in order to for the SIM to determine it a qualifying referral.

Client Intake Additional Info - Default Loading Questions: Select the way you want the program to present the custom questions in the window.
