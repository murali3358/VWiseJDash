PIR Control Screen

This screen is used to create a summary transaction for each child that qualifies for the PIR Counting Tool. This file stores much information to speed up the processing time of the overall counting while all Users are accessing the tool.   In order for a child's application to be on the Attn 1 Day list, the child must have been enrolled in a classroom and also must have at least one PRESENT attendance posting completed during the program year. 

Records can be reviewed in this screen by Show Attn 1 Day or Show All Apps.  The User selects the radio button filter at the top of the screen that the system will using when calling up the records for review.

  • Show Attn 1 Day will call up every child that has been enrolled in the PY and has had attendance posted for at least one day.  (the standard requirements for a child to be included in the PIR Report)
  • Show All Apps will clal up every child that has an application entered in the PY with or without being enrolled or having had attendance posted for at least one day.

Of more importance, this Screen controls the applications that are updated in the PIR Counting Tool.  This control must be updated by the User prior to reviewing the PIR Counting Tool to ensure that the numbers that are reviewed are as current as possible.  (new enrollment or attendance records are added in.)

Let's understand that!  If the User who enrolls children in the system enrolls a child on Monday morning and then a User at the Center posts attendance at the center the PIR Control must be run in order for the system to find the NEW enrollment and attendance data.  Once the system finds that child's application all of the data that the PIR Counting Tool picks up for that child will now be picked up when the PIR Counting Tool generates counts. 

How does a User run the PIR Control?  Easy.  The User Generates the Work Data and then Creates the Work File.  This process takes seconds and will become a standard process for Users that are interacting with the PIR Counting Tool. 

Locate the Generate Work Data and Create Work File buttons in the lower left hand corner of the screen.

These two buttons are presented together to ensure that from the beginning a User understands that BOTH processes must be completed to ensure that the data reviewed in the PIR Counting Tool is as current as possible.  This process takes seconds!  Work from left to right. 

  • Click Generate Work Data button
  • Click Create Work File button
  • Click OK
  • Done!

Any User that works with the PIR Counting Tool should create a routine habit of updating the PIR Control to ensure the records are updated and counts are as current as possible.  If a User thinks a particular PIR question count is incorrect there is every potential that a PIR Control record update will produce different data for review!  (Maybe a child was recently enrolled or another User just posted attendance.)

Defining Funds in the PIR Control Screen

The  button that is located in the top right hand corner of the PIR Counting Tool is where the system administrator will tell the system how to sort the enrollment funds into the PIR Counting Tool.  A left mouse click access the working screen below:

Left Mouse Click the TYPE column in order to activate the cell in the list box to enter the HS or EHS fund type for all Fund Descriptions that you want to be a part of the PIR reporting.  Some programs have just HS or HS and EHS as a Fund Description.  Other programs have an extensive list like we see in the example above. 

These definitions can be changed/updated at any time but typically are kept in place once they are set in place.
