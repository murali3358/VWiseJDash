Create a New User

  1. Click ADD   at the top of the Assign User Account & Privileges tab to activate ADD mode.
  2. Customize the User Account by selecting the controls that will allow the User to interact with the system as needed in the screen below. 
  3. Click  SAVE

 

  • User Id - Use a unique code. 
  • Caseworker - Use a unique code.  This is used in reports as an identifier
  • Image Types - Check the box(es) to assign viewing privileges to uploaded images attached to applications.  This table code is customized in Agency Table Codes. 
  • Staff Member - Select the staff member in the drop down.
  • Security - Combination hierarchy access allows one User to access different levels of security in the system.
    • Reception/Clearical - Basic Intake / cannot code or access hierarchy secret applications.
    • Case Manager - Basic Intake / can code or access hieararchy secret applications.
    • Program Administrator - cannot code secret / can access hierarchy secret applications / can change caseworker.
  • EMS - If the User is interacting with the Emergency Services Module this is where you assign the appropriate supervision level that you want the User to have. 
  • Site - Assign a User access to a specific Site.  (i.e., In Head Start when a User is assigned to a specific Center, assign that Site.) 
  • Active - A User that can access the system with their UserName and Password.
  • Inactive - A User that you no longer want them to be able to access the system with their UserName and Password.  Do not DELETE Users, make them Inactive. 
  • Access All - This selection is only give to System Administrators in order to access all User accounts as needed.  This is a confidential access to all accounts that should be given to Users with full User privilieges and very high security access only.  When using this login level of access the system administrator can interact with the system as any other User.  Contact CAP Systems for the password procedure to use. 
  • Template User - A specific User can be designated a Template User in order to assign global User privileges to other Users.  This allows you to create a complex User and copy it to other Users in the future.  Check this box if you want to be able to copy the User privileges to another User.  The copy function does not connect the two Users together.  If you make adjustments to a template User it does not change any other Users previously copied. 
  • Force Password Change - Use this if you want the User to have to change their password the next time they log into the system.  Also, use this is a User forgets their password.  Reset it and the User can login as if they were a brand new User and change their password again. 
  • Copy From - When adding a NEW User you can select Copy From and select the User you want to copy.  The User you want to copy must already be a template User. 
  • Set the default hierarchy that will load when the User logs into the system.  The User will always be able to change the hierarchy during a work session by using either the current hierarchy button or the default hierarchy button in the advanced search screen when necessary.  The decision to be made in setting up the new User account is what is the most obvious hierarchy to set for the default.

NOTE**** - User Account passwords are CASE SENSTIVE.   Brand New User Accounts are uppercase for both the UserID and the password. 

Edit a New User

  • Click EDIT at the top of the Assign User Account & Privileges tab to activate EDIT mode. 
  • Edit the User Account by changing the controls that will allow the User to interact with the system as needed.
  • Reset Password - this button now replaces Copy From.   
  • Click SAVE
