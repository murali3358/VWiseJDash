This report should be run just before bundling.  It allows the User who is posting invoices to the system to verify their work, before bundling, before check processing.  The report verifies all the invoices put in by allowing the User to look through the list.   

When the User runs the report, they will get a dollar amount.   If the dollar amount is what was expected, this is an additional checks/balance with bundling. 

If the dollar amount is not what was expected there is the opportunity to make adjustments before bundling. 

Note – if there are open invoices for deliverables and the User creates utility or rent and the User creates a Utility or Rent bundle those will appear in the invoice validation. 

Deliverables with no autorizations:  Yes, gives a list of all deliverable clients that have not been authorized yet.  No, bypasses that option.

Authorization with no Invoice:  Yes, Deliverables clients that have authorizations out there, but there have been no invoices as of yet.  No, bypasses that option.

Unpaid Payments:  BRIAN

Summary:  BRIAN

Sort by:  Select Applicant Number, Client Name or Vendor

Start Date:  Always use the first date of the program year.  Just like in bundling!

End Date:  Current date is the default and will work just fine

User last updated by:  All Users is the default.

Vendor:  All produces all data.  You can run this report by Vendor.  It’s a matter of preference.
