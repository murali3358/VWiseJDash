List Open Bundles:   (default filter when the screen opens)

Make sure when you look at the bundle you can confirm the total dollars and bundle type is the bundle you want to work with.

Once the bundle has been created the Users comes here to close the bundle or remove the bundle.

To close the bundle, check off the select check box and then left mouse click CLOSE BUNDLES Button.

Preview Button allows the User to access the report/preview maintenance screen if you need to reference an existing report. 

 

To remove the bundle, check off the check box and then RIGHT CLICK in the row and you will see the option to remove.   Left Mouse click the blue box to the left of Remove Bundle to remove it.  

 

 

Once a User closes a bundle, the system moves the record to the List closed Bundles group.  The User can’t see this happen as its behind the scenes.  Selecting the List Closed Bundles filter will present all closed bundles. 

List closed Bundles: 

This is where the User is getting ready to do the physical check run.  (check processing)

Confirm the selected bundle is the correct bundle you want to work with. 

Step #1:  Select Bundle

Before generating work file, the User can open the bundle in this screen.  This will send the bundle back to the List Open Bundles Group.

Step #2  Generate Work File

When that happens the system will present all applications included in the bundle for final verification.  If you don’t like something you see CLEAR WORK GRID.  This allows the User to return to the bundle that was selected.  The only option when there is an error in the bundle is to REMOVE the bundle and go to the Control Card and fix the invoice(s). 

The system auto fills the Enter the starting check# (1080) but the User must make sure it’s correct.  Confirm the starting check number using your agency protocols. 

Then check the date (it defaults to the current date) change it if you want.

Starting Voucher is the number the system will auto increase, by one, from the last one it used.   There is no confirming necessary.

Payment # the system also keeps track of this and auto fills it. 

  • Click Generate Work file
  • Confirm check date and totals
  • Click Process Reporting Button which will produce the check pay report, the check register and the physical checks.  (even if you don’t produce checks from CAPTAIN).  
  • When the system has completed all those tasks, the PREVIEW button appears.  Click that to review.

 You will be looking for reports that look like these below:

Ck_checkpayreport_payment#_checkdate

Ck_checkregister_

Ck_checks_

 

 

If the User goes to the Control Card for an application included in the bundle the check date and check # for the invoice will now show it has been paid. 

 

List Paid Bundles: 

This lists every bundle that was physically paid.  Preview gives the details of the selected bundle.  Users commonly use this filter to confirm a bundle # and what was paid out. 

 

List Payment #’s

Some customers have multiple bundles paid under one Pay Number.   This filter allows the User to select the Pay# and re-run the check processing step.  (typically this event occurs when a check jams in the printer)

Don’t forget to confirm all fields at the bottom.

Generate Work File allows the User to review applications included in the Pay #, and if confirmation is correct process reporting (to get a new check pay report, check registers, new checks). 

 

Click Preview and then go and print out check pay, check registers, and print checks (if you use CAPTAIN). 

 

How to do a Check Processing Re-Run

 

Step 1:  select List Payment# S radio button

Step 2:  select the applicable Pay# (remember Pay# may not be the same as Bundle #)

Step 3:  this step will depend on what went wrong.  Did the user enter the wrong check # and/or check Date or did the checks jam?

·         Wrong check # just enter the new check number in the Enter the Starting Check # field

·         Wrong Date just enter the correct date

·         Checks Jam just enter the new check number and make sure the date is correct

Step 4:  Click on ‘Generate Work File’ button

Step 5:  Click on “Process Reporting”

 

 

The user will now see RR in front of the check reports (they will not be with all the others because of the RR in front as everything in the Preview comes down alphabetically).  The user should preview them to verify everything is correct before they proceed.  Also you can see the original Check Reports are still there so it is up to the user what they should do.  If it was me I would delete them so that I wouldn’t open or bring a check register/ check payreport or checks that are wrong.  But again that is up to the user and the agency.
