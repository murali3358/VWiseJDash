Program Definition

This screen allows you to define all of the general information associated to the program and customize specific actions or references in the system.   

  • Assign Poverty Guideline Charts
  • Build Eligibility Criteria
  • Control overall functionality for the program.  (ie - auto refer, edit Case Type, No Income Type)

Take the time to understand this screen and how it controls the other screens in the system.  Fill in the Director and Contact Person information to ensure full integration with the SIM/ Benefits Bank. The System Administrator must be familiar with the features to fully understand the purpose of all the data fields in the system.

When adding a NEW program to a hierarchy, this ADD screen above allows you to assign the program to a specific hierarchy.

  • Hierarchy -  Click the  to assign the program to a specific hierarchy.
  • Program - Name the program as you want it to appear across the system.  This is what all Users will see when selecting what hierarchies they want to select to work.
  • Short Name - An abbreviated version of the program name.  This appears in various tempalte reports as an identifier.
  • Year -  This field is used to segregate data by program year. If the program year is not applicable leave the year field blank. This approach is for Case Management you do NOT use this field leave it blank. 

    Head Start and Energy Services MUST use this field in order to manage the data correctly with the specific application.  Enter the PY that you started using the system.  

    For example:  Head Start reporting year for the PIR is Sept2010-June2011.  This is the 2010 program year in the system.  Enter 2010 in this field.
