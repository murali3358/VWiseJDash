Performance Measures Tab

The Performance Measures Report delivers an unduplicated count of services delivered.  During the construction of the report template, the User selects the Group Defintion(s) that they want to see displayed in the report.  All services related to the group defintion(s) selected will be produced when they report is run.

This Tab allows the User to create any service groups that need to be used during Performance Measures Report building. 

Groups are created and then Tables are attached to the Groups.  Any critical activity that is active in the system will be available for selection when buildng your groups. 

 

The Tab has 3 distinct sections:

  • Reference Period
  • Group Descriptions
  • Table Descriptions

Let's look at them.

Reference Period in the Performance Measures Tab

The Reference Period is the date range that reflects the full reporting cycle for the Group Definitions you are building.  When you select the reference period in the report the system will create a start date and a stop date and only interact with data that falls into the reference period. 

CSBG/ROMA data management is a very straight forward example to use.  You can apply the represented standards of buildng reference periods for any other reference period grouping you need. 

If your CSBG reporting year is 10/1/2012 - 9/30/2013, you will need to have a reference period in the system for that date range with all the Group Definitions you want to review in reporting. 

Locate and click on the ADD Button in the main Tab toolbar at the top of the Master Tables Maintenance Tab to access the working screen to add a NEW Reference Period to the system. 

Let's look at the working screen.

  • Active Check Box - allows you to see and select the reference period in the system.
  • Copy From Check Box - allows you to copy an expired reference period without having to design it from the beginning again.  Select the reference period you want to copy from the drop down option field. 
  • Click SAVE.

When the working screen closes you will find the new reference period in the list box with the others. 

Group Defintions in the Performance Measures Tab

The Group Definition section is where you will build ANY Group Definition you want to be able to select during report building.  (See Performance Measures Report.)  Group Definitions are defined in the reference period where they are going to be used.  It's easy to think of it this way.  You build the group definition once.  In the first reference period you want to use it in.  In subsequent reference periods it will be copied.  So, you only have to build a group defintion once!

Locate and Click the ADD   Button in the row of the reference period to access the working screen you will use to define the group.   The column in the Reference Period List Box is ADD GROUP.

Let's look at the working screen.

  • Group Code - this is a unique code you assign to the group definition.  It can be visually meaningful or simply a unique numeric code.  The system uses this during report building.  Users do not need to identify or use it.  The system will use it. 
  • Group Description - this should be a meaningful, brief description to clearly identify the group to Users for selection in report building.
  • Result Headers - These are the header identifiers in the report template.  The system loads in the headers you assign to the group definitions.  A suggestion to visit the Result/Status drop down options you have built into the system.  You will want to build headers that are relevant to them.  You can combine more than one status/result response to the same result header. 
  • Achieved Result Column # - typically this is the first column, but you can assign any position within the Headers that you want. 
  • While all fields are required, you do NOT have to use all the result headers as long as you use one. 
  • Don't forget to click SAVE to lock in your new group definition.

When you close the working screen you will then see the newly build Group Description in the Group Description List Box.

Any EDITS or DELETIONS you need to make to the Group Definition is done from the right hand side of the row in the Group Defintion List Box.

Table Definitions in the Performance Measures Tab

The Table Definition section is where you will build all Table Definitions that you want included in Group Definitions sections of your report building.   (See Performance Measures Report.)   This is where you will go and find the critical activities that are being tracked in individual applications and lock them into the Group Definition where you want the number to be in the report.

Table Descriptions have a 1:1 relationship for each critical activity.   While you can create any table description you would like, duplicating the critical activity for effective report identification is the best practice.  It is also recommended that you use the critical actiivty unique code as the unique code in this section.  Bottom line, everything that is custom built stays together and it easily identified regardless of the area of the system you are in. 

Locate and Click the ADD   Button in the row of the group definition to access the working screen you will use to define the table.   The column in the Reference Period List Box is ADD TABLE.

Table Descriptions put into place specific tasks the system performs when producing a report.  How counts are aggregated is one very important process.

Let's look at the working screen.

  • Table Code - a unique code for the system to identify the table description. 
  • Count Indicator - select the methodology you want the system to use to produce the unduplicated count for the report.
  • Table Description - a brief description of the critical activity.  Best Practice is to duplicate the critical activity.
  • #Expected to Achieve - full number expected to achieve the critical activity.  The system will calculate the percentage achieved for the date range of the report based on this number.
  • Disabled - this will disable the table definition.  It will be inactive, though not deleted.
  • Duplicated - this will produce a duplicated count in the report. 
  • Age From - this is an additional filter to limit the data to the age range input. 

Don't forget to click SAVE to lock in the table definition in the system.  When you close the working screen you will find the table description in the Table Description List Box. 

Goal/Service Associations in the Performance Measures Tab

Click on the  button to open a working screen to create the Goal/Service Assocations in the Performance Measure Report Template. 

This step comes after you have created the Tables.    By creating each individual association, once, Users will be able to run a Performance Measures Report and get the specific data counts that were built in the Group Definitions. 

This association is typically 1:1, but can be 1 to many. 

Click Goal/Service Associations Button in order to access the screen to create and save the association(s).  Below is what that screen looks like.

  • You can do associations one by one OR you can stay on this screen and select Group and Table over and over again creating the association(s) you need. 
  • Set the Goal Field for the Group you created in the last screen that you want to work with.
  • Set the Table you created in the last screen that you want to work with.
  • FIND and Highlight the specific Goal from the grid list that you want the system to associate.
  • Click CHANGE
  • Click SAVE
