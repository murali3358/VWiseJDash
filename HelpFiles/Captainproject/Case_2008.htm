Fields Control what does that really mean?  While your Users are completing data entry they will be moving from Screen to Screen.  They will move from Tab to Tab.  As well, they will move from field to field in Screens and Tabs.  Which fields they can interact with are determined by the Fields Control Screen.

Fields Control Maintenance Screen allows the System Administrator to control what fields will be Enabled, Required and Shared throughout the system.  The system works with the following field attributes:

  • Enabled - this field is available to the User for data entry.  The User can enter information into this field or select drop down options.
  • Disabled - this field is not available to the User for data entry.  These fields will be greyed out and the User will not be able to interact with it.
  • Required - this field is required and must be answered in order for the User to save their work.  This field is denoted in the system by a red asterisk. 
  • Shared - When an application has been entered into the system and is subsequently drawn into another hierarchy position, all existing data in shared fields will follow the application to the new position. 

Field attributes can be assigned to **ALL Hierarchies and then minor changes made to the few select hierarchies that need individualized options. Hierarchy changes will override any previous ALL field attributes for that hierarchy. 

You have the ability to control the field attributes in the following screens:

  • Client Intake
  • Income Entry
  • Income Verification
  • Contact Posting
  • Critical Posting
  • Milestone Posting
  • Medical Emergency 

****Note there are two tabs in this screen:

  • Field Control Assignment - this tab is where you interact with field attributes
  • Custom Fields Definition - this tab is where you have the ability to design custom fields and assign them to the hierarchies that need to work with them
