Once a User Account has been created you must visit each tab along the bottom of the screen to add/edit specific hierarchies and/or privileges assigned to the selected User.  You will need to activiate the ADD  button or EDIT  at the very top of the screen (above the tab label) to begin your tasks. 

Inside each tab you will have the ADD  button or the EDIT  button available to work within EACH TAB.  Add presents the very first time you are working with a New User.  Edit presents in subsequent visits so that you can edit existing privileges that you have already assigned. 

*Note - When you are working with the individual tabs clicking the add/edit button on the right hand side opens up a new screen to select the specific functions the tab has to offer.  If you do not activate add/edit mode the system displays the existing privileges for reivew only. 

User Details Tab

View Only displays the User details for the selected User.  Add/Edit mode allows you to add/edit User details.

Intake Hierarchies Tab

Boxes checked in this tab determine the specific hierarchies the User can access when working with intake screens.

Service Hierarchies Tab

Boxes check in this tab determine the specific hierarchies the User can access when working with services.

Screen Privileges Tab

Boxes checked in this tab determine whether the User can view/add/change/delete.  You can assign any combination of access to each screen.  

Report Privileges Tab

Boxes checked in this tab determine which report screens the User can access and what they can do while using the screen.

 

Additional Privileges Tab

The drop-down fields allow the System Administrator to select all the applicable component access that the User needs. These Component Tracks are accessed at the Client Tracking window in the Head Start Module.  The field selections are driven from the agency table COMPONENTS.
