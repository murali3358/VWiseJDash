Verify Income in the Client Intake Screen - REQUIRED

Verifying income is a crucial step in Client Intake data management.  This ensures that reporting will include all the necessary information required.  Every application in the system MUST have income verified.

Note*  When copying an application from one hierarchy position to another, income verification does not copy over with the Client Intake data.  The User must re-verify income after copying an application.  If not, the reports will not include the necessary data from the applications that are missing income verification.  The system will prompt the User with a RED FONT message at the top of the Intake that income has changed and must be re-verified.

Access the Income Verification Screen  in the Client Intake Screen 

Click on the Button that is underneath the household members grid box at the top of the Client Intake Tab.

 

 Complete the Income Verification Screen 

  1. Check the Verification Date check box and enter the date that the income was verified for the application you are working with.
  2. Select the Verifier,if required
  3. Select the Funding Source, if required by program definition (i.e., Head Start Users will do this)
  4. Check all the boxes that apply for the Income Verified section
  5. Select Meal eligibility category, if required by program definition (i.e., Head Start Users will do this)
  6. If there is a Categorical selection, complete that.  The default is none.
  7. If the program will reverify income at a specific interval, check the box for Reverification Date and enter the date the income should be reverified.
  8. Click SAVE

Once you click save the next available row in the top grid box will populate with all the selections you made before saving.  Income can be verified as many times as needed. 

Verifications serve as a historical path of income changes in the application.  Users should NOT delete income verifications if they are no longer current/correct.  Users to simply add a new verificaiton with the date used and then the application will hold both reference points of income verification for anyone to review. 

When is it appropriate to DELETE an income verification row?   Only when the User realizes immediately after completing a verification that is is incorrect.  If the User does not have delete privileges they will need to contact their system adminstrator to complete the deletion.  

The Income and poverty guidelines percentages will automatically populate in the income verification screen.  If a chart box does not populate with a specific percentage that indicates there is NOT a chart available to the system to use to calculate that.  If you feel a particular box should populate, contact your system administrator so they can update the system as needed.
