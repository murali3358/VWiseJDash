MainMenu Search is the User's Gateway into the sytem.  The starting point. 

This screen does not close but instead acts as the foundation for User activity.  If a User returns to MainMenu Search and searches for a new application, and selects it, any open screens will automatically display the newly selected application records. 

CAPTAIN manages the data Users enter in many different ways.  The system has been built to make decisions when generating reports.  The decisions the system makes are based on the data that was input as well as the way the data was input. 

Searching for a Client in the system before entering data in the system is crucial to generating accurate reports when you need them.  The system is designed to prompt a User to search for the client they are working with before they will be able to access the ADD function and move forward. 

Search for a Client in the MainMenu Search Screen

If you know the Application Number of the client you want to find you can enter that into the Applicant # field at the top of the MainMenu Search Screen and hit enter.  The system will load that application and display all the household members in the application for review.  (customers typically update data entry forms to include a place to note the application number)

You will note that the system has loaded the primary client into the top of the MainMenu Search Screen and the hierarchy location where the application has been saved.  This is telling us that the system has found application #6691 and loaded it into position for us to work with it.  We can now select any screen from the Tree on the left hand side and review or work with the contents of application #6691.  (i.e., clicking on Client Intake will move you to the Client Intake Screens)

Remember - Once you select an application from the search results the system holds that in the MainMenu Search.  Every Screen or Tab that you open continues to present all the associated data to the application that the system is locked on.  In addition - you must return to the MainMenu Search Screen and select another application to work with.  Any existing Screens or Tabs that are open will automatically shift to the newly selected application.  

Use Advanced Search for a Client in the MainMenu Search Screen

If you do not know the specific application # you need to work with, click the Advanced Search Button in order to have access to several expanded search options. See the full Advanced Search Screen below.

The search results will display in the search box.  Let's review the column titles.

  1. Hierarchy - The hierarchy position where the application has been saved
  2. Year - if the program has been defined to use year you will see which year the application was saved.  Energy and Head Start are two typical programs that manage data by program year.  If the program year definition isn't in place the column will be empty. 
  3. AppNo#- The number the system assigned to the application when it was entered and saved.  (A client can be listed in more than one hierarchy with more than one application number.)
  4. SS# - displays the last 4 digits of the social security number.
  5. Phone - phone number entered into the client intake screen
  6. Address - Address entered into the client intake screen
  7. Case - The module location where the application was entered and saved. 

Change the Hierarchy in the MainMenu Search Screen

Changing the hierarchy at the top of the search screen dictates where the system looks for the search criteria you have used.  Checking the Search Entire Database Button tells the system to search the entire system. (the hierarchy position at the top of the screen is ignored.)

Results generated from the search will display in the search box.  When there is more than one row of search criteria you will be able to select a row to put the system's focus on that application to begin your work.

Set as Current Hierarchy in the MainMenu Search Screen

When a User has access to more than one hierarchy, a current hierarchy can be selected.  Setting the current hierachy tells the system that during the logged in session the User is working the current hierarchy will load into the MainMenu.  This current setting does not change the Default Hierarchy.  In fact, the next time the User logs in the system will move off of the Current Hierarchy and load in the Default Hierarchy. 

The Current Hierarchy can be changed as many times as the User needs during the logged in session. 

Set as Default Hierarchy in the MainMenu Search Screen

When a User has access to more than one hierarchy, a default hierarchy can be selected.  Setting the default hierarchy tells the system which hierarchy to load into the MainMenu when the User has logged into the system. 

The default hierarchy can be changed as many times as the User needs. 

Work with Search Criteria Fields in the MainMenu Search Screen

The Search Criteria Fields allow you to filter search results to minimize the volume of data that will display for your reivew. 

  1. Search By - select the filter you want to use. 
  2. First Name/Last Name - will be available when searching by name.  When using these fields the system will search for the exact criteria that you input in these fields.  It will display ALL the resulted criteria it finds, including expanded names that have the search criteria within it.  (i.e., SMI will yield SMITH, SMITHSON, etc.)
  3. Search For - select the filter you want to use
