When adding a client application to the System, the Client Intake Screen is the first area you will complete.

Your System Administrator controls all required or enabled fields.  Data collection that is required for specific FUNDING SOURCE REPORTS should always be required. 

As some families are reluctant to share their social security numbers, when a client does not provide you with their SSN please press the Page Down key while locked on that field so the System will assign a Pseudo SSN.  A User should never make up a SSN.  This automated assignment ensures that there is never a duplicate pseudo number in the database. 

The NA check box should be used when the DOB isn’t known or it hasn’t happened yet.  This override should not be used for the standard approach to entering applications.  The system has the ability to determine the age of each household member at Intake and also current age based on any date selected.  If the DOB has not been entered this will not function as needed.
