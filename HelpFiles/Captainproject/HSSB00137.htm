Attendance Posting by Site

Attendance Posting by Site screen allows you to manage all attendance and meal records for all children enrolled your program.  This feature makes managing attendance and meal records FAST and EASY.  This screen is designed so that the User never has to leave the screen in order to affect all the data entry using their own preferred filters.

What that means is a User can post for just one day, or post for an entire week during one posting session.  All data entry can be completed and then one SAVE to be done.

IMPORTANT NOTE**** - The very first posting that is completed for a child is a PRESENT.  Never post an ABSENT before a PRESENT.  The system calculates dates of entry and other important details that are connected to attendance.  In addition, once attendance has been posted the ADA Report picks up all posting records. 

When you select Attendance Posting by Site in the screen tree this screen tab opens:

This is a read only screen until you activate it by clicking the  (Add Button) or the  (Edit Button).   Many times a User only wants to access a class list and ulitmately attendance or meal records to reference them.  The ability to review saved records in read only mode ensures that no changes are accidentally made by a User who may not be trained in the attendance posting data entry process. 

Setting Filters in Attendance Posting by Site

The fields at the top of the screen allow the User to individualize their experience.  Filters can eliminate records you don't need to see or allow you to create the specific posting list you want to work with.

  • The Class Field is where you select the specific classroom that you want to load into the screen to review or complete data entry.  The selection options the User has are based on the site and room maintenance structure in the system. 
  • The Sort On Field allows you to sort the child names in the processed list by first name or by last name.  Most Users find selecting the sort on filter that matches your attendance sheet most effecient.
  • The From Field and the To Field is where you set the date range for the processed list you want to review or work with to complete data entry.
  • The Previous and Next 10 days Buttons moves the FROM/TO date range back or forward 10 days without the User having to change the dates in the two fields. 
  • The Status radial selection produces saved records for the children with that status.  As a new User it is more efficient to select BOTH until you determine what is the best approach for your needs.  Note**** PostAll is disabled in BOTH so attendance records cannot be posted to a withdrawn child.   Selecting both ensures you see ALL attendance/meal records available for review.  Enrolled shows you only currently enrolled chidlren and Withdrawn shows you only withdrawn children. 

Process a Class List in Attendance Posting by Site

Once the User sets the desired filters in place, clicking the PROCESS BUTTON will prompt the system to process a class list and load it into the screen.  If there are attendance/meal records saved the screen will load in that as well.  The image below is an example of a class list for one day with the attendance and meal records saved. 

If there are no attendance/meal records saved for the date (or date range) in the filter the children's names will load in without any checked boxes.  Let's take a look at the columns in the attendance posting screen to understand their function.

  • The first column on the left shows a GREEN "P".  This identifies to the User that attendance has been posted.  If it's blank, no attendance has been posted for that date.
  • The second column identifies if the child was enrolled or withdrawn.  This visually assists the User in knowing whether attendance should be posted on that date or not.
  • Child Name is displayed based on the Sort On filter selected. 
  • The Date column displays the date that the row is locked on.  If your FROM/TO date range is greater than one day, there will be a row for each date included in the processed list.
  • The Day column can be helpful when processing a list for multiple days.  It's helps the User find a day of the week instead of focusing on the date.
  • The next three check boxes labeled P, A, and LA are where you check the condition of that dates attendance.  Was the child Present, Absent or do you need to account for a Legal Absence.  Check the appropriate box based on the child's attendance for that date.
  • The Reason column is where you select the absence reason code when you have selected the absent check box.  Right Mouse click in the cell field for the child/date you want to enter and select the absence code you want. 
  • The BM column (Bill Meals) is a checkbox that toggles the meal selections designed in the system on or off.  When the User selects P (for Present) the system will automatically populate the breakfast, AM Snack, Lunch, PM Snack and Supper (BALPS) columns with the correct meal pattern.  If a child is absent the User must uncheck the NM button.  Once this is done the system will automatically deselect the meal pattern to match the absent posting.  If you need to uncheck just one meal and not another, for a specific date, you can do that by left mouse clicking the checked box you want to remove. 
  • FUND, Parent Fee, Fund Fee, and Hours Spent are columns that are populated by the system based on the design in site and room maintenance. 
  • The Post All Button allows the User to post a present to ALL children in the list with one click.  This is a time saving feature that can be used even if all children were not present.  Important to note - if the User has selected more than one date in the FROM/TO date range clicking the Post ALL button will post a present to ALL child records for ALL dates.  The easiest approach for accuracy and easy review is to process a list for only one day at a time to use the Post All feature.   This feature is disabled when the User selects to review both enrolled and withdrawn child records in the list box.  This ensures that attendance records cannot be posted to any dates when a child was withdrawn. 
  • SAVE and CANCEL Buttons allows the User to saved the attendance/meal records entered or leave the screen without any changes, respectively.  IMPORTANT!  The most common mistake new Users make is they post their attendance/meal records and forget to click SAVE.  Once you have changed the date range and processed a new list everything is lost unless it has been saved BEFORE you process the next list.
