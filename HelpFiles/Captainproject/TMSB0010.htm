The Bundling Report screen is used to create a bundle prior to doing a check run. 

 

 

Create New Bundle:  Yes/Print Existing Bundle -  Yes must be selected to create a new bundle.   Print Existing, a sub screen will show all bundles that have been created.  The User can reprint a previous bundle. 

Clients:  The default is ALL.    Funding Source options in the drop down create a bundle by that specific funding source. 

Vendor:  The default is ALL.  The User can pick one specific Vendor if needed and create a bundle.

Dollar to Pay:  this filter allows the User to set a cap on the total bundle value by entering that in the field.

Process:   Utilities, Deliverables, Rent.  Deliverables is the default.  This filter dictates what bundle is being completed.  One at a time, only.

Start Date:  This date always has to be from the beginning of the program year.  Day #1 of the program year is what the User inputs there.   No Exceptions!

End Date:  this field defaults to the current date and should not be changed.

It is very important to review and verify that the bundle is correct.  The review is to confirm the dollar amount is what was expected.  Double Check before running checks. 

The Print Existing Bundle filter allows the User to print an existing bundle in the system.
