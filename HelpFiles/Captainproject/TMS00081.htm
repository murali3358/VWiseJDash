The Control Card manages all aspects of the LIHEAP process from benefit awards to authorizations of fuel deliveries and to initiate payments to vendors. This screen performs all 3 functions and is broken up into 3 areas.

  •  Benefit Determination
  •  Authorization of Deliverable fuels
  •   Invoicing of Deliverable fuels

All aspects of the client LIHEAP benefits and invoices can be managed thru this screen.

 

Benefit Maintenance - Determination

 

This is where the certification is done for each household.  Without a benefit there cannot be any authorizations or invoices.  If adding a benefit the User must click on the Add button in the tool bar which will open the Benefit Awards panel.

In the Benefit Awards panel there is an icon where Users are able to access the Income Details screen and post income for all Energy Intakes.   

Remember the income type for the member may have been previously entered in the Client Intake Income Details panel.  The User must click on the ‘add/edit’ button in order to select Income Type.

Right click on an Income Type row in order to create a second row for the same income type.  This will allow the User to add additional income for an income type that has already been used. 

After the Benefit record is created the User can print the letter from within the Benefit Sub Panel.

Points to consider …

  • Income should always be verified before the user certifies.
  • Click on the Certify Process button which will enable all the required fields **allows the User to start the certification process (** fields are always enabled)
  •  Date field the User will not be allowed to enter a date that does not fall within the program year.  And when a re-certification is done the date will automatically change to the current date.
  • Now enter the information (such as date; assets; income certificate, etc.)  The Case Worker field will automatically default to the user log in.   Heat Source will be captured from the Client Intake in the housing expense tab.  If this is not correct the User must go back to the Client Intake screen and make the change BEFORE they certify.
  • Go back to the client Intake Housing Exp & Services Inquiring Tab and complete the middle section with the correct information.

 

  • Once all the information has been added the user MUST click on the “calculate award” button so that the system will calculate whether or not the client is eligible and if eligible the system will calculate the benefit.

  • If the Vendor is not correct, the User needs to go back to Client Intake and click on the SUPPLIER Button

The Supplier VIEW screen allows the User to make any necessary corrections to the Vendor. 

Once all the information has been added the user MUST click on the “calculate award” button so that the system will calculate whether or not the client is eligible.  If eligible the system will calculate the benefit.

NON-Qualified Aliens the system will NOT count that member in the household size field but if there is income that income will be used to calculate the eligibility of the household.

Crisis awards and Safety net awards … these awards are issued to the deliverable fuel clients after their basic benefit issues is used up. The ‘C’risis award is automatically made available to the client but the ‘S’afety net award is after a Risk Assessment form is filled out.  The number of safety net awards made available to the customer is controlled by DSS.

 

 

Insufficient Documents: This control will allow the user to flag the clients record as Denied for Insufficiant Documentation. If the proper documentation is submitted at a later date the record can then be certified with a benefit.

 

Authorization of Deliverable Fuels

Click on the ’add authorization’ button to add an authorization. User must fill out all the required fields.  Then press the save button. And once an authorization has an Invoice associated to it the user will not be allowed to ‘edit authorization’.

 

  • In order for an Deliverable Authorization to be entered into the system the County Differential MUST be entered into the system by the agencies System Administrator.
  • Authorizations, Crisis and Safety Nets will not be allowed if the date does not fall within the specific program date range.
  • The number of Safety Nets and Safety Net Benefits are now controlled by a table and not within the control card.

 

Invoicing of Deliverable Fuels

 

  • The User must select the applicable Authorization and then click the Invoice radio button.  Now the User can add an invoice.
  • If the invoice exceeds the authorization the system will automatically reduce the amount.
  • If the User has noticed that they have entered the wrong gallons, delivery date they must press the ‘change’ button to fix whatever is wrong.  As well, the User must tab through the rest of the fields before they press the ‘save’ button.  This is so the system will recalculate the invoice correctly.
  • When the invoice is entered the ‘differential’ field will be filled and that differential will be added to the Harbor before comparing the Harbor Cost from the retail.
