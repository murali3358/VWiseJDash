Eligibility Groups allow the System to manage a much more complex eligibility model for Program Eligibility and Referrals.

The Group Description should be meaningful.  It should tell the reader what the group of questions directly relates to.  It should also allow the reader to determine which group of eligiblity criteria they are looking at. 

For example:

Head Start Income Eligiblity and Head Start Categorical Eligibility are two distinctly different assessments that are reivewed in order to determine if a client is eligible for program services.  The User familiar with the program services would understand the details of both groups. 

Let's look at the Group Description Screen we will work with.

 

Click the Add  Button in order to access the working group description screen to add a new description. 

This is the screen that opens:

 

  • The system will present the hierarchy in the top field that this new Group Description will be added to
  • Type a brief, meaningful group description that best tells the reader what the group is for.  This is particularly important if there is more than one group description
  • Points - you can assign a point value to this group that will be referenced in the SIM.
  • Conjunction -  criteria condition conjunctions cannot be combined.  Mapping out the AND conjunctions and the OR conjunctions requires the System Administrator to create two groups with different criteria.
