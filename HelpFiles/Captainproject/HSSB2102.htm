Track File Lists Report Screen

Track File Lists Report will show the User how the Master Track and Component Tracks are designed.  Content Area Specialists can quickly and easily use this as a checklist of what is currently available to be tracked and monitored in the system.  System Administrators can use this report as checklist when double checking that all tracking information has been associated to the PIR Counting Tool.

The User can generate a full Master Track component list, select just one specific component track or ask the system to deliver ALL the tracks designed in the system.
