The SIM (Service Integration Matrix) Feature is the foundation for doing presumptive eligibility for any program in the system. This feature looks at the intake data entered into one department intake combined with custom questions (if designed and answered) that are relative to eligibility criteria for any programs in order to create a referral record. 
 
 
Where does the SIM data come from?

The Client Intake Screen!  Different fields throughout Client Intake may factor into a program's eligibility criteria.  Your system adminstrator will have designed the eligibility criteria for programs which is the reference the SIM uses when determining a potential referral.  If you will recall during the Client Intake entry process there were questions in the Intake Details Tab of the Client Intake Screen.  In addition to standard intake data fields, the responses to those questions will factor into the referral records that will be created when the SIM is run. 
 
  • The System creates the record for the referral in the SIM (Service Integration Matrix) Screen. 
  • In order for SIM feature to deliver the maximum information the User should be certain to complete the Question Section of the Client Intake data entry for all household members. 
  • Questions can be revisited at any time, and then the SIM can be run again to update any new/additional program referral records
