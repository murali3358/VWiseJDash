Income Details in the Client Intake Screen - completed AFTER entering all household members

Now we have our Client Intake full of household members and it's time to attach income to each of them.  This area of the application can change during the time that you are serving the household.  Income Changes.   New income can come into the household.  Existing income can be lost.  The income details that are attached to all household members is used to determine the Fed OMB % of the household, specifically where the household falls on the poverty guidelines chart.  When income changes have been updated in the system it is important to re-verify income. (See Verify Income. )

Locate and click the to access the full screen to enter income.

View Income Details in the Client Intake Screen

Let's look at an image of what we can see when we click on the Income Details Button in VIEW mode. 

  • The system only presents rows completed with income information. 
  • There is a .pdf button in the top right hand corner to print the income details if you need that. 
  • Remember the system standard of selecting the household member in the top left hand corner in order to view their income records.

Add Income Details in the Client Intake Screen

Locate and click the button to access the working screen to add income details for any household member.

Let's look at an image of the Income Details working screen. 

The top left-hand side of the Income Details Screen presents all the household members entered in the application.  As you enter income the Total Income and Program Income cells will update.  All of the data in this box connects directly to the Client Intake Screen.  When you return to the Client Intake the same data will be there. 

  • It is important that you select the household member you want to attach income to and confirm that the arrow is on the left hand side of the row their name is in AND the cell is BLUE. 
  • This tells you that when you enter income in the box section of the screen it will be attached/associated to the correct household member. 
  • If you don't confirm this you run the risk of attaching income to the wrong household member. 
  • It also allows you to easily update income as you continue to work with your client. 
  • If a household member leaves the system will adjust income quickly and easily as long as the income is distributed correctly when entered. 

The top right-hand section of this screen asks for name of the person that verified the income (select the name) and then enter how the income was verfied (W2, Tax Forms, Letter, etc.) In addition, you can add the income source. 

  • You'll note that in Add/Edit mode the system now presents all the income types available to attach to the selected household member. 
  • If income details have already been entered they will present at the top of the list box.
  • There is a .pdf button in the top right hand corner to print the income details if you need that. 

  • If you don't find an income type in the system that you need to use, contact your System Administrator

Let's look at the row of income cells so you understand what each column is doing. 

  1. Income Type - select the specific type of income you are working with
  2. Interval - the time interval you want the system to use when calculating the income entry  (Monthly has two different monthly fields the system will consider.  The second monthly field is used for bonus/commissions/etc. that are part of monthly income but not the household members actual wages.)
  3. Exclude Check Box - check this box if you want to system to IGNORE this income when calculating program eligibility. 
  4. Amt1 & Date1 - these cells work together.  You'll also notice some cells are PINK.  When you select the Interval the system immediately PINKS the cells you MUST complete.  Enter the interval amount of income and the date that applies to it.  Depending on the interval you are working with you may need to enter more than one set of amount/date. 
  5. Factor, Sub, and Total - these fields are controlled by the system and update according to the selections you make while entering income. 
  6. Delete - quick and unforgiving.  If you click delete the system will remove the income entry you were locked on.  Confirm before deletion. 

 

Edit Income Details in the Add/Edit Working Screen

Locate and click the same button to access the working screen to edit income details for any household member.

  • You can update income, add addtional income, delete income, using the same approach you did for adding income. 
  • If you need to update an income row click on the data you want to update. 
  • If you return to this working screen to edit/update income you must remember to re-verify income.  (See Verify Income)

Add an additional row of a specific income type in the Add/Edit Screen

This feature allows the User to enter two income types with the same description.  Let's say you have a household member that works two different jobs.  You then need to attach two separate WAGES to that household member.  You can add Wages for the first job, add a new row, and then add Wages (for the 2nd job) again.  Both rows will fall together in the list box. 

  • Locate the specific Income Type you want to work with
  • Right Click on the Income Type Cell
  • Click the message that says Add Additional Income Type Row
  • The message box disappears and you will now see an additional row inserted for the specific income type you selected
  • Click on the new row and enter your data

 

Delete Income Details in the Client Intake Screen

Income details are deleted, by row, by specific income type.  This allows Users to make any necessary adjustments to just one Income Type in an application.  Scroll the Add/Edit Screen all the way to the far right.  At the end of each row you will find a DELETE  Button.  The delete button function is row specific.  Click the delete button in the row of income details you want to delete.

Deleting income details is permanent and cannot be recovered once it is done.  Double check that you have selected the correct row before you confirm to the system you want to delete.
