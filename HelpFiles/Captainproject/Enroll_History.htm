Enroll History Screen

Enroll History allows the User to review the enrollment history of a selected child as well as manage enrollment events.   There are five distinct sections in this screen:

  • Current Fund Status - the current enrollment information for the selected application (One current status per FUND. For example, if you have HS and EHS FUND you can have a child enrolled in EHS and on the Wait List for HS.  Two current records, one for each FUND.)
  • Status Timeline - this timeline shows the User what has happened for the selected child prior to the Current FUND Status
  • History - the history area is read only.  It alllows the User to see what changes/updates have been made to the selected application.
  • Staff Members and Primary Position presents the staff to client associations for the selected application if this feature is used by your program
  • Head Start Rankings are found at the bottom of the history screen.  These rankings are system-driven based on the ranking design the system administrator has put in place. 

Often times when reviewing data records a User will want to be able to see this enrollment history!  This screen is dynamic and updates as the enrollment data changes. 

In addition the system manages an AS OF timeline that is used in reporting and the enrollment/withdrawal screen.   It is important to consider every status change and every date change as EVERY change will impact the status timeline and will impact AS OF date data that is working for you behind the scenes. 

If the Current Fund Status has a record(s) the User can access applicable functions to update the current status of the application by right mouse clicking the status row they want to work with.  A Right Mouse Click on the record you want to work with will give the User access to the menu for actionable changes/updates. 

Let's look at that right click menu:

Edit Status in Enroll History Screen

Selecting Edit Status in the right click menu allows the User to edit the current enrollment status ONLY.   When a child is enrolled in a SITE/ROOM there is a date attached to that.  Users should NOT edit the status timeline with dates AFTER that current enrollment date.  Another way to look at this is that history dates and current status dates should never overlap.  (If this overlap occurs, the only alternative is to delete all attendance associated to the status, delete the status, and then re-post the attendance.  Taking a print screen of posted attendances allows the User to have an accurate reference for adding the attendance records back in. This can only be achieved by a User with Program Aministrator security privileges.)

The working screen below will allow the User to edit the current fund status. 

Add Fund in Enroll History Screen

Add Fund allows the User to add a new fund into the child's application. 

Edit Fund Date/Details in Enroll History Screen

Edit Fund Date/Details is where the User will access a screen to change information already saved.  Thisis the area to change the date attached to a specific enrollment record or change the FUND supporting the services the child is receiving. 

Enroll Note 1 & 2 act like a sticky note on the record.  The User can input a brief description that will follow the record in various locations.  It can be helpful to others Users to know why the edit was completed.  It can also be a helpful reminder to the User that made the edit when reviewing the record at a later date. 

 

Transfer in Enroll History Screen

Users can transfer ANY status.  The example we have used here is a Wait List child; however, if the child was enrolled the steps would be the same.  The selected row in the current fund status dictates to the system which record you want to transfer. 

The Enroll History Screen that is found in the system standard buttons at the top of the Enrollment/Withdrawal Screen allows the User to make changes to the records attached to a selected child.  This screen is accessed AFTER selecting a SITE/ROOM, loading in the list of available children and then selecting the child to work with.  This sub-screen is associated to the child selected in the Enrollment/Withdrawal list. 

In the image below we can see the example child is currently on the Wait List.  When the User RIGHT CLICKS the Wait List Status column (In the Current Fund Status section) they can select from the menu the action they want to complete.  For this example, we would select TRANSFER in order to transfer the child from the Wait List at the assigned Site/Room to another Site/Room within the site structure for the program year. 

 

Once the User selects the transfer action the following sub-screen opens:

The data that is already associated to this child's record will automatically load into the screen. That information will be a part of the transfer action, creating a status timeline record at the same time.

Complete the applicable fields and then click the TRANSFER button.  This will transfer the child to the selected SITE/ROOM on the date entered.  You will also see what was the current record prior to the click fall into the timeline status for future reference. 

Delete Status Record in Enroll History Screen

Delete Status Record is only available in the Enroll History Screen .  In addition, the function is only available to Users that have Program Administrator security rights.  Other Users will not be able to see this option in the action menu.  Users that do not have this security access should contact their system administrator to affect the necessary deletion they need.

If there are attendance records attached to a status, the system will not allow the User to complete the deletion.  The User will receive a message identifying that there are attendance records attached to the status.   The User must then delete the attendance records and then return to complete the status deletion. 

This should be done with caution and care.  If a User deletes attendance records this will impact previous USDA meal reporting.  Meal counts are within the attendance records.  (Capturing an image of the attendance records you are deleting, will allow you to add them back without any errors.)

Denied App# in Enroll History Screen - Case Management Module ONLY

Denied App#    - this status change is used in the Case Managment Module ONLY to clearly indicate in the SIM referral process that the Intake is now a Denied App.  This option does not present in the Head Start Module as it is not a part of the enrollment or withdrawal process for status changes.
