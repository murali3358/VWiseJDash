Add a New Applicant  from the MainMenu Search Screen

In order to add a new application to the system you must first complete a search to determine if the client you are working with is already in the system.  (See MainMenu Search )  The system will NOT allow you to enter a brand new application without first confirming that the client is not already in the system.

When you have searched the system for the client you want to work with and do not find them in the system you can immediately start entering an application from the search screen.

Locate the  in the lower left hand corner of the MainMenu Search Screen.  Click that to enter the Client Intake Screen. 

There will be 3 Tabs to work with.  Let's look at an image of that.

 

  • Applicant Details - this section is where you will add demographic data about the client you are working with
  • Intake Details - this section is where you will add in the address, phone number
  • Employment Information - this section is where the employment data is entered

Based on the program definitions and fields control set in place by your system adminstrator you should complete all available fields in these tabs. 

Also, the bottom section of these Tabs presents questions that will allow the system to determine if the client you are inputting is eligible for any other services.  ( See SIM - Service Integration Matrix )

Applicant Details Tab in the Client Intake Screen

Let's look at an image of this tab.

Straight forward data entry!  Click and Pick it.  Set it and Forget it.  Touch data once and then all Users can interact with it for various reports or application review. 

Remember the system standards. 

  • If the field is required you will see a RED asterisk next to the field literal.  Those fields must be completed in order to save the application. 
  • PgDn key will assign a Pseudo social security number of you don't have it.  Do NOT create your own Pseudo SSN#.  Let the system go to work for you.  The system will never duplicate a Psuedo SSN# to ensure that unduplicated reporting functions correctly. 
  • Screens/Tabs are in VIEW mode.  You must go into ADD or EDIT mode to add or update existing information.
  • Click SAVE before moving off the Tabs or you work will be lost.

Active Check Box - Unchecking this box indicates that this individual was once a part of the family but is no longer a member due to a death, divorce, moved out, etc.  You do not want to delete this household member from the application because it reflects the family's changes to its make-up during the entire service history.

Secret Check Box - Checking this box will make the application secret so that other Users cannot see the Intake.  This must be used with caution!  Only Users coded as CASE MANAGERS will be able to mark an application Secret.  Once an application is marked SECRET only that User will be able to uncheck the Secret field.  One Case Manager cannot see another CASE Managers secret application(s).

Users coded BOTH PA and CM will be able to see ALL secret applications; however, they will not be able to uncheck the secret box.  That still stays tied directly to the User that made the application secret.   

Exclude Member -  Checking this box tells the system that this household member is still a part of the family but for a programmatic eligibility reason(s) s/he is not to be counted in the household total for the program. 

Now we move on to Intake Details!

Intake Details in the Client Intake Screen

Let's look at an image of this Tab.

CASE TYPE - this field is crucial to successful reporting.   Your system default is CASE MANAGEMENT CLIENT.  All clients served should have this Case Type tag.  Rule of Thumb - don't change it!  In order for the system to accurately aggregate data counts for the Case Demographics Report this needs to be consistent.  The system can only run a report on one Case Type.  Therefore, if all clients are entered with the same Case Type you can always see all the data, agency-wide when needed.

In the middle section of this Tab there are three date fields:

  • Initial Date - optional, can be used as a pre-intake date measurement if needed
  • Intake Date - required, it is the date that you started working with your client
  • Case Review Date - optional, if you have a future date that you want to review the application input that here

SITE - this field allows you to look at data by location.  Maybe you deliver the same service in 5 locations and want to ability to see data by each site.  Your system adminisrator will have set the SITE tags in place.  If you don't see what you think you need, contact your system adminstrator to request an addition to the Site List.

Assign Alert Code in the Client Intake Screen

There are times that a User will want to indicate information about a client via the alert codes for other staff to be aware of.  An excellent example is DEAF.  If a client is deaf or even hearing impaired it will be valuable information for other staff viewing the intake. 

All client based areas will have this alert code available.

You access the available alert codes by clicking on the  at the very top of the Client Intake Tab.   Your system administrator controls the selections available in the Alert Codes Screen.  If you don't find what you need contact your system adminstrator and request an addition be made. 

Once you access that button you will find the options available to select.

  2. Click OK once you have made all the applicable selections

When you return to the Client Intake Tab, you will see the check box(es) selected to alert anyone viewing the application of valuable information they should note.

Add Case Notes in the Client Intake Screen

Case Notes allows any User that has access to the screen to review additional information that another User has attached to the application.  Let's call it a system Post-it Note.  Often times there is important information that isn't managed within the structured fields of the different screens. 

Case Notes are accessed through the Library Book  Button at the very top of the Client Intake Screen above the Tabs. 

Click on the Library Book  Button to access the Case Notes Screen.

When you see the Libary Book  Button with the magnifiying glass that is an indicator that a User has entered a Case Note that should be reviewed. 

ADD - To add a Case Note(s) entry to an application.  Click SAVE to attached it to the application. 

CHANGE - To edit a Case Notes entry, click SAVE to keep your changes.  This is the function you use to remove a specific part of the Case Notes.  (i.e, if you have 3 different entries and want to remove just one of the entries.)

DELETE - To delete the entire Case Note(s) that is attached to the application.  This is permanent.  The Case Notes cannot be recovered.

PRINT/VIEW - This allows the User to see what the Case Notes will look like when they are printed.

PRINT - This function allows the User to print the Case Note(s). 

CLOSE - This closes the Case Notes Screen when the User is done reviewing what has been entered.
