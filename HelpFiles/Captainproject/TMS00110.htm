This screen shows the User all the SITE calendars that have been built in the system.

 

SITE Code – unless the sys admn has assigned a specific site code to a User, it will always be blank.  If it’s blank the User needs to search for the SITE they are going to book an appointment. 

Year – always stays in CURRENT program year. 

Month – select month you want to book the appointment in

Week Day – ALL , brings up all the SITE calendars that have been created.  Selecting a specific weekday will filter the list box to include ONLY the weekday you selected.  See the example below:

Schedule Date field allows the User to select one specific date.  The list box will present all SITE calendars in the system for that date. 

The Quick Launch button will open the Appointment Scheduling EDIT screen for the date that is in the Schedule Date field.  It's a quick access left mouse click to get to the appointment scheduling screen without filtering or scrolling through the entire list. 

In the list box:

  • Available Slots column decreases as appointments are scheduled. 
  • Status (open or closed) tells the User whether the day can have scheduled appointments or not.
  • If you want to schedule an appointment outside of the current date (a future date) highlight the specific month/day row in the list box and that will take you to the selected date to schedule an appointment. 

The Appointment Schedule working EDIT screen:

 

  • The SITE will automatically fill if the SITE is assigned to the User.  (this is completed by your Sys Admin in the assigned user privilege screen).  Assigned sites will auto fill.  If the User has NO site assigned to their User, this field will be blank.  The User must first select the SITE they want to work with. 
  • The User can change the date they want to work with in the DATE Field.
  • Refresh Schedule must be completed if the date is changed. 
  • Next Available Slot – will take the User to the next available slot, regardless of the date. 
  • Search by Name/SSN – verify that the person doesn’t have an appointment.  Enter the Uesers name or SSN.  This is a tool to confirm if the client already has an appointment. 
  • Each row for each date will accept a scheduled appointment.  Add, Edit, Delete in the bottom.  All SAVED records will present in the list box. 
  • Certified Approved already for this year, the system will present a message telling the User they are already in the system. 
  • HEAT Source -  how the client heats their house.
  • INCOME Source – select ALL applicable incomes. 
  • WORKER – resource your sys administrator for your agency protocols for using the worker field.  The system will always save the User that booked the appointment in the Person field in the list box.
