Let's look at the Fields Control Maintenance Screen Tabs 

Directly above the two screen tabs you see the Screen Name Field.  When you select a screen from the Screen Name field the system will display the existing hierarchy controls in place.  

Add Fields Control to a specific hierarchy in the Fields Control Maintenance Screen

Let's look at the Field Control Assignment Tab first.  You must manage screen controls by hierarchy.  Setting controls for all hierarchies puts a set of controls in place across the system.  There are many fields that will be consistent across all programs.  Once ALL hierarchies controls are in place you can go in and custom design specific program hierarchies that need something just a little bit different.  (i.e., maybe a particular program doesn't need one field enabled that all the other programs do.)

Subsequent edits to original controls, by individual hierarchy, will override the ALL hierarchies controls in place.  

The ADD  Button above the Fields Control Maintenance Tab confirms with this mouse-over message:  Define Field Controls.  Locate and Click this button to access the Field Control Assignment working Tab below.

  • Select the screen name you want work with in the Screen Name field
  • Select the hierarchy you want to add controls for in the top right hand corner by clicking the hierarchy  button
  • Field Description identifies the field in the system
  • Sub-Screen points you to the location of the field in the system

The check boxes are optional:

  • Field Validation - if this box is checked the User will receive a message if they tab through a field that is required.  The User must enter a response in order to continue.
  • Enable all Fields - enables all the available fields in the screen.  This can be used as a starting point particularly when you know you want to disable just a few fields.  It's more time efficient to enable them all and then disable a few then to check through each field and enable them.
  • Share all - Sharing a field ensures that if an application is copied and saved into a new hierarchy position that all the field data collected from the copied application will transfer to the new application.  If a User copies an application and finds that a required field is empty, check fields control to confirm if the field should be shared.  Then the data will transfer from one application to another.

In the absence of checking any of the check boxes, you can interact with individual field controls row by row.  Locate the field you want to work with and check the box to activate the column designated control. 

The button must be clicked in order for the system to make any adjustments to the field controls in the hierarchy you are working with.  It's a good idea to return to the screen and confirm the controls you want in place have been realized.  Sometimes what you think you put in place and what actually happened are different. 

 

Edit Field Controls to a specific hierarchy in the Fields Control Maintenance Screen

Left Mouse click the specific hierarchy that you want to work with.  The blue highlight indicates which row the system is focused on.  Find and selection the specific hierarchy field controls set you want to work with. 

Click the EDIT  Button to access the controls you want to change/update.
