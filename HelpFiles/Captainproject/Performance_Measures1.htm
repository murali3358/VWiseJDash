Performance Measures Report Screen

This report generates an unduplicated count of the services delivered in a specific data range.  This report template interacts with the Group Definitions designed in the Master Table List.   The default reference period date follows the current CSBG funding year.   The system will not allow a system administrator to design reference periods that overlap.   

  • Make certain that the hierarchy selection is correct at the top of the screen.
  • Case Type - Universally across your system all Users select the same case type when inputting an application for a client or the entire household.  Select the Case Type used.
  • Case Status - set the filter that you need the system to work with
  • Date Selection - Select the date the milestone was achieved or the date the data was entered in the system.
  • Reference Period - the full reporting year relative to the report your are running. 
  • Report Period - input the date range (within the reference period) that you want to see the data records for.  It can be as small a date range as you would like as long as it does not exceed the reference period.
  • Site - the filter allows you to look at services by SITE assigned during data entry.
  • Categories - this filter will produce counts for associated goals while eliminating unassociated goal lines from the report completely. 
  • Produce Details - filter your report to produce details or not.  Not sure?  Run both and see which works best for you!
  • Poverty Level - this filter allows you to eliminate certain poverty levels from the report by setting a level range. 
  • Groups - in the sub panel, select the groups you want the system to use when building the report.  (Groups are built in relations to Reference Periods in the system.  See your System Adminstrator if you're not sure what has been built)
  • Report Format - select the filter you want the system to use when generating the report?  Do you just want to see the Performance Measures or do you want to see the Performance Measure plus the goals.  Not sure?  Run both and see which works best for you!
  • Secret Applications -  include secret applications in the report calculations or not.  Select the filter you want for your report. 

When this report run is completed the User will find unduplicated counts organized by Group Defintions.  Group Definitions are designed by your System Administrator in  Master Table Maintenance.  Contact your system administrator for more information about that.
