Case Demographics Report Screen

This report delivers to the User the case demographics for clients/household members that received services in a given date range.  This report template interacts with Case Demographics, in the Master Table List, and presents all qualifying Client Intakes in the report. 

  • Make certain that the hierarchy selection is correct at the top of the screen.
  • Case Type - Universally across your system all Users select the same case type when inputting an application for a client or the entire household.  Select the Case Type used.
  • Funding Source - select ALL or filter to a specific FUND
  • Case Status - set the filter that you need the system to work with
  • Milestones/Services - select the one you want in the report run
  • Date Selection- this filter will look at the date of the service in applications or the date the data entry was completed.
  • Reference Period - the full reporting year relative to the report your are running. 
  • Report Period - input the date range (within the reference period) that you want to see the data for.  It can be as small a date range as you would like as long as it does not exceed the reference period.
  • Site -the filter allows you to look at services by SITE assigned during data entry.
  • Attributes -this filter selects the Income Type Category presentation you need for your report.  (User Defined is used for the CSBG/IS Report.)  Income and No Income may not equal the total households.  If one household has a HHL MBR with none as income and another with at least one income type posted, that household will be counted in both categories.   In order for this to balance the entire system approach must be that income is only entered NONE when there is no income in the household.  That NONE should be attached to the primary client only. 
  • Produce Statistical Report- Yes filter will produce additional reports (3) that will show the User how the counts were aggregated. No - there will be only one (1) additional report generated that will show the User what data was excluded from the report. 
  • Poverty Level -  this filter allows you to eliminate certain poverty levels from the report by setting a level range. 
  • Zip Codes - filter your report to review data only in specific zip codes
  • County - filter your report to reivew data only in specific counties
  • Secret Applications -  include secret applications in the report calculations or not.  Select the filter you want for your report.
  • Demographic Counts - the filter will present data only for clients receiving the services or every household member in each application. 

This report run will present a report that has two columns.  The left-hand column will show the report period count.  The right-hand column will show the cumulative count for the entire reference period.   This report template design allows the User to review a period of time within the full reporting year and review both counts in the same report run.

Let's discuss how the system runs this report and why the reference period (in relationship to the report period) is so very important!

The Report delivers to the User an unduplicated count of clients benefiting from services/milestones.   Establishing a full reporting year reference period is the first important step.  The next important step is not stepping outside of that reference period throughout the entire reporting year.  Why are those two steps important to success reporting counts? 

The system always starts at the beginning of the reference period.,  If you don't consistently use the same reference period (regardless of your reporting period) you will not get accurate unduplicated counts for subsequent reporting periods.  The system looks through the reference period to determine if the client with posted services/milestones was already counted outside of the reporting period (yet within the reference period).  If that is the case, the system will skip over the reporting period services/milestones and not count them.
