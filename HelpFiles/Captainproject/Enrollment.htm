Enrollment/Withdrawals Screen

This screen is where Users manage enrollment and withdrawal events and all the other actions that surround those.  In order to begin you will first want to select the specific SITE that you want to interact with in the top drop down field.  Once you have made that selection the list box will load with all corresponding rooms assigned to that Site.  Next, select the specific classroom you want to work with.  You do this by left mouse clicking the row of the classroom you're selecting. 

**** is the ALL Rooms position and is NOT an actual classroom.  Some programs assign wait listed children to the SITE and not a specific classroom.  This position in the list box for each SITE allows the User to do that.  The counts in each category reflect total SITE counts in this row as well.

ADD status record function is the ONLY function that must be completed in the Enrollment Withdrawal Screen, selecting the application and clicking ADD.  All other enrollment movements/functions/changes/updates happen in the Enroll History screen.  That screen can be accessed independent of the enrollment withdrawal screen or from within the enrollment withdrawal screen.

  • From and To  date fields will populate with the current program year's open and close dates the system resources from the program definitions the system adminstrator has put in place in the master file.
  • As of Date field is where you will tell the system the cut off date you want the system to use when calling up records for review.  This allows the User to have the list reflect the actual enrollment list, or wait list, as of a specific date. 
  • Status allows you to filter the list to the specific status condition you want to work with.  (ie - the print screen above will generate a list of records of all children with the as of date taken into consideration.  Changing the STATUS field to WITHDRAWN would then only present the children that have been withdrawn.)

Counts/Room Details button allows the User to review the demographic makeup of the classroom they are working with based on current enrollment records at that time. 

SHOW drop down field allows the User to filter the application records that will populate the list box.  This gives the User a very dynamic experience when working with enrollment tasks.  All Sites/no status records allows the User to load into the list box child applications that may have been assigned to another SITE but now the enrollment task needs to be assigned to the SITE selected. 

Selecting an application by checking the box directly to the left of the child in the list will allow you to make edits and updates.  Once you've checked the box, move your mouse to the top of the tabs in your browser and click EDIT to open the screen below.

  • Date - Enter the date you want for this record transaction
  • Status - Select the Status you want to use for this record transaction
  • Desc 1 & Desc2 - Descriptions allow the User to make a few notes specific to the record transaction.  Another User that calls up this record can read whatever was added there.

Add Child(ren) to the Wait List

The User will want to select the SITE in the image below, where the child was assigned in Client Intake.   The concept is find the child in the SITE list where they are being tracked.   The User will also use the SHOW field to filter the list that will populate in the List Box with all qualifying child Intakes to work with.

 

  • Once the child Intake has been located in the list box, the User must check the check box on the left hand side of that row.  Left Mouse Click checks the box. 
  • Then the User must click the ADD button at the very top of the screen to access the working screen to place the selected child on the Wait List

  • Status - select Wait List
  • SITE - the default is the SITE the User is locked on.  Accessing the hierarchy funnel will allow the User to assign the child to a specific ROOM wait list.
  • Date - the date the child is being placed on the wait list
  • Fund - the fund supporting this child's participation in the program
  • Enroll Note1 & Enroll Note2 - two text fields where the User can enter a brief note of explanation that ALL Users can see.  A point of reference.  An additional reason or additional piece of information that could be important when reviewing the enrollment history
  • Fund Category - the fund supporting this child's participation in the program
  • Fund End Date -  if the fund supporting is related to a specific time frame that is not open-ended, enter the end date here.  If it's ongoing year after year, a future year far enough out that the life of the child's participation in the program will end before the end date arrives. 

These next three field are used if you are using our Accounts Receivable Module where the system is calculating DC bills for fees.  HS program services are free and unless you have extended day care where there is a billing process these fields are not used. 

  • Rate Effective -  date the rate goes into effect
  • Parent Daily Rate -  the family's co-pay that they are responsible to pay if their household is receiving subsidized day care
  • Funding Daily Rate - the daily rate to be used in calculating the day care bill.

After completing all the fields the User clicks CREATE STATUS RECORD.  This click actually places the child on the Wait List. 

 

Transfer a Wait List child in Enroll History Screen

The Enroll History Screen that is found in the system standard buttons at the top of the Enrollment/Withdrawal Screen allows the User to make changes to the records attached to a selected child.  This screen is accessed AFTER selecting a SITE/ROOM, loading in the list of available children and then selecting the child to work with.  This sub-screen is associated to the child selected in the Enrollment/Withdrawal list. 

In the image below we can see the example child is currently on the Wait List.  When the User RIGHT CLICKS the Wait List Status column (In the Current Fund Status section) they can select from the menu the action they want to complete.  For this example, we would select TRANSFER in order to transfer the child from the Wait List at the assigned Site/Room to another Site/Room within the site structure for the program year. 

 

Once the User selects the transfer action the following sub-screen opens:

The data that is already associated to this child's record will automatically load into the screen. That information will be a part of the transfer action, creating a timeline history of the movement that is taking place.

Complete the applicable fields and then click the TRANSFER button.  This will transfer the child to the selected SITE/ROOM on the date entered.  You will also see what was the current record prior to the click fall into the timeline status for future reference.
